RUNO

An online multiplayer clone of the popular card game UNO

RUNO is a browser-based multiplayer online game modeled after the popular card game UNO. I built this project in order to get additional hands-on practice covering the new skills I have developed thus far while working through the Full Stack Web Developer Nanodegree and Front-End Web Developer Nanodegree programs offered by Udacity. Here are some examples of problems that I had to solve to successfully complete this project:

  • Perform all game logic on the server side in Python
  • Transmit game state, in JSON, between server and clients
  • Frequently update and persist game state data
  • Synchronize back-end game state file IO
  • Employ object-oriented JavaScript techniques to manage the client-side representations of game objects
  • Manage many dynamic UI elements with jQuery
  • Implement flash messaging system to keep players updated
  • Create a responsive front-end so the game plays well across a wide variety of devices
  • Utilize CSS transitions to create fluid movements

I thoroughly enjoyed working through the challenges that this project presented. It forced me to tap the many skills I've gathered in my full stack studies as well as my front-end studies. I hope you round up some opponents and give the game a whirl. Enjoy!

By completing this game project I believe I have demonstrated my ability to effectively write object-oriented JavaScript software, process and render graphics using the HTML5 canvas API, as well as handle user input events. Project #3 for Udacity's Full-Stack Web Developer Nanodegree

Prior to starting this project I completed two Udacity full-stack web developer courses: "Full Stack Foundations" and "Authentication and Authorization with OAuth". I thoroughly enjoyed learning how to use the Flask framework as well as SQLAlchemy. The power and flexibility of this stack is immense. Throughout these courses I learned how to do the following:

  • Create a web application using Flask
  • Harness the power of Flask's features
  • Create and interact with databases using SQLAlchemy
  • Utilize Google's OAuth2 API to implement an authentication system
  • Implement an authorization system
  • Build a RESTful web application
  • Implement CRUD (create, read, update, delete) methods
  • Build forms for user input
  • Perform client side and server side input validation
  • Process image file uploads
  • Prevent cross-site request forgery (CSRF) attacks
  • Use other OAuth providers such as Facebook

I had a blast completing this project. I decided to implement a simple music shop website which allows users to log in and create, edit and delete items. Users log in with their Google account and must be verified by an administrator (me) before they have write access to the site. Once enabled, they can create items and then edit or delete those items. Admin accounts can edit or delete any item on the site. Since I'm also working through Udacity's Front-End Web Developer Nanodegree program I decided to push myself to implement a responsive front-end and give it some polish. Check out this live link from your phone, tablet, laptop and desktop systems to see how the layout responds. After finishing this project I now have the necessary set of skills to dig deeper into the Flask framework and SQLAlchemy module to create more complex web applications.
